1. ПОСТАНОВКА ЗАДАЧІ
   Розробити бібліотеку для роботи з графічними форматами (BMP) на мовах C та C++.
   Бібліотека повинна забезпечувати:

- Створення, читання та запис файлів.
- Малювання примітивів (лінія, прямокутник, коло).
- Виведення інформації про файл та його вмісту (ASCII-арт).
- Демонстрацію ідентичності роботи алгоритмів на C та C++ (cross-test).

2. ОПИС ФОРМАТУ BMP
   BMP (Bitmap) — це растровий формат зберігання зображень без стиснення (у даній реалізації).
   Файл складається з:

- BITMAPFILEHEADER (14 байт): сигнатура 'BM', розмір файлу, зміщення даних.
- BITMAPINFOHEADER (40 байт): ширина, висота, бітність (24 bits = RGB).
- Дані пікселів: масив байтів у форматі BGR.
  Особливість формату: кожен рядок пікселів має бути вирівняний по межі 4 байт (padding).

3. РЕАЛІЗАЦІЯ
   3.1. Мова C (структурний підхід)

- Використовується структура `Image` для зберігання даних у пам'яті (через `malloc`).
- Функція `save_bmp` реалізує серіалізацію заголовків та обчислення padding:
  Formula: pad = (4 - (width \* 3) % 4) % 4.
- Функція `read_bmp` десеріалізує бінарний файл у структуру.
- Функція `print_ascii_art` конвертує яскравість пікселів у символи для відображення в консолі.

  3.2. Мова C++ (ООП підхід)

- Клас `BmpImage` інкапсулює ширину, висоту та вектор пікселів `std::vector<Color>`.
- Реалізовано принцип RAII для автоматичного управління пам'яттю.
- Використано потоки `std::ifstream` / `std::ofstream` для вводу-виводу.

  3.3. Алгоритми малювання
  Для малювання ліній та кіл використано алгоритм Брезенхема (Bresenham's algorithm).
  Це цілочисельний алгоритм, який уникає повільних операцій з плаваючою комою,
  що забезпечує високу швидкодію.

4. ТЕСТУВАННЯ

- Unit-тести: перевірка створення та малювання фігур.
- ASCII-візуалізація: перевірка завантаження зображення у консоль.
- Cross-Test: автоматичне порівняння бінарних файлів, згенерованих C та C++ версіями.
  Результат: файли ідентичні, тест пройдено.
